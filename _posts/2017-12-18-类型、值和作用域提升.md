## 一、类型

### 1.1 内置类型（除对象外，其他统称为“基本类型”）

**注1：可以用typeof运算符来查看值的类型，返回的是类型的字符串值。**

**注2：js中变量是没有类型的，只有值才有.**

> js一共有7种内置类型：
>    + 空值(null)
>    + 未定义(undefined)
>   + 布尔值(boolean)
>   + 数字(number)
>   + 字符串(string)
>   + 对象(object)
>   + 符号(symbol, es6中新增)

## 二、值

### 2.1 字符串和数组

> js中基本类型的字符串是没有属性和方法的，但是字符串经过包装（即包装一个封装对象）之后，变成字符串对象，那么就具有一些和数组类似的成员函数了。

##### 字符串和数组的区别

> js中字符串的成员函数不会改变字符串本身，而是创建并返回一个新的字符串。

> 而数组的成员函数都是在它的原始值上操作。

> 还一个不同点在于**字符串反转**。数组有一个字符串没有的成员函数reverse().要解决这个问题，有一个方法是先把字符串变为数组，反转之后，再把数组变回字符串。即：

```
str.split("").reverse().join("");
```

> String#trim()方法：去掉字符串前后的空格，返回新的字符串。

### 2.2 数字

> 数字可以用toFixed()方法指定小数部分显示的位数。如:

```
var num = 42.59;

num.toFixed(0); // 43
num.toFixed(1); // 42.6
num.toFixed(3); // 42.590
```

### 2.3 null和undefined

> 这两个一个表示空值，一个表示为定义。**但是，null为一个特殊的关键字，不是标识符，不能把它当做变量来使用和赋值。另外，undefined为一个标识符，可以被当做变量来使用和赋值**。如：

```
undefined = 2;
```

## 三、作用域提升

> 作用域这个东西说简单，但是又总是容易出点错。这里，我们只讨论一个**提升**。

先甩出一句话：

**包括变量和函数在内的所有声明，都会在任何代码被执行前首先被处理，并且函数首先被提升，然后才是变量。**

### 3.1、小思考

看到**var a = 2**;会想到什么？

一般呢都会认为这个是一个声明，就一个赋值语句，把2赋值给a。

但是，js其实会把他分成两个部分：**var a；和a = 2；。第一个定义声明是在编译阶段进行的，而第二个赋值声明会被留在原地等待执行，即代码执行前就进行了var a。** 

看下面一段代码：
```
console.log(a); // 这里不会报错而是会出现undefined，因为，后面的var a；在编译阶段就已经声明了，此时已经有了a，只是还没给a赋值而已。
var a = 2;
```
再来一段代码
```
1. foo(); // TypeError
2. bar(); // ReferenceError
3.
4. var foo = function() {};

解析：这里第一句是TypeError表明foo已经声明了的，知识foo不是函数，才出现了类型错误，而bar根本没声明。具体理解，就是下面这样

var foo; // 声明被提升到了作用域的顶端

foo(); // 声明的foo没说是个函数，所以这个TypeError
bar();

foo = function() {};
```

### 3.2 函数声明和变量声明的优先级

由于函数声明和变量声明都会被提升，所以很明显就想到一个问题，到底是哪个会被提升到最上层呢？**其实，函数会首先被提升，然后才是变量**。看代码:

```
foo(); // 输出1。即函数的声明在变量声明前面。

var foo;

function foo() {console.log(1);};
```
